# Migration Guide: v0.6 to v0.7

This release is mostly transparent — your workflow code runs unchanged. The only action required is removing calls to two store methods that were removed.

## Breaking Changes

### 1. `WorkflowStore.UpdateRunStatus` removed

This method was never called by the engine. Remove any direct calls.

```go
// Before — remove this
err := store.UpdateRunStatus(ctx, runID, gorkflow.RunStatusCompleted, nil)

// After — use UpdateRun instead (already how the engine works)
run.Status = gorkflow.RunStatusCompleted
err := store.UpdateRun(ctx, run)
```

### 2. `WorkflowStore.CountRunsByStatus` removed

```go
// Before — remove this
count, err := store.CountRunsByStatus(ctx, resourceID, gorkflow.RunStatusRunning)

// After — use ListRuns and count yourself
runs, err := store.ListRuns(ctx, gorkflow.RunFilter{
    ResourceID: resourceID,
    Status:     gorkflow.RunStatusRunning,
})
count := len(runs)
```

### 3. `ExecutionConfig.FallbackStepID` removed

This field was never read by the engine. Remove it from any config structs you built manually.

```go
// Before — remove the field
cfg := gorkflow.ExecutionConfig{
    MaxConcurrency: 4,
    FallbackStepID: &someID, // remove this
}

// After
cfg := gorkflow.ExecutionConfig{
    MaxConcurrency: 4,
}
```

---

## What Changed (No Action Required)

### True parallel execution

Steps declared with `.Parallel(...)` now actually run concurrently. Previously they ran sequentially despite the declaration. No code changes needed — workflows are faster automatically.

### Cancellation works correctly

`engine.Cancel(runID)` now immediately signals the running goroutine to stop. Previously it only wrote to the database and the goroutine kept running. Cancelled workflows now always end with `RunStatusCancelled` (not `RunStatusFailed`).

### Context propagates to store calls

Per-step timeouts now also apply to state reads/writes inside the step. No code changes needed.

## Migration Checklist

- [ ] Remove any calls to `store.UpdateRunStatus(...)`
- [ ] Remove any calls to `store.CountRunsByStatus(...)`
- [ ] Remove `FallbackStepID` from any `ExecutionConfig` literals
- [ ] Run `go build ./...` to confirm no compile errors
